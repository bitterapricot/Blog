PicGo + Gitee 实现图床

发表于 2022-06-11 更新于 2022-06-12

图床是在线图片的个人仓库，通过图床图片链接可实现图片访问。

下文介绍PicGo + Gitee 实现个人免费图床，并在Typora（Markdown编辑器）中进行应用。

Gitee 注册账号并创建仓库

Gitee(码云)是OSCHINA.NET 推出的代码托管平台,支持 Git 和 SVN,提供免费的私有仓库托管。

注册账号

Gitee官网，注册账号

创建仓库

点右上角“+”号选择【新建仓库】

输入仓库参数（选择“开源”）【创建】

注意：创库地址在PicGo设置中需要。

创建Gitee私人令牌

注意：令牌在PicGo设置需要。

PicGo 安装及设置

PicGo是一个用于快速上传图片并获取图片 URL 链接的工具。PicGo是一款开源软件，可直接下载安装包（建议选择最新版本，本文以2.3.0-beta.6版演示），或下载源码。

PicGo 安装

  1. 运行PicGo安装包，安装PicGo

  2. 安装PicGo插件，图中安装了任意一个即可

注意：插件安装后，退出重启PicGo应用

PicGo 设置

【图床设置】》【gitee】或【插件设置】点击插件设置按扭选择【配置 upload - gitee】

设置Gitee仓库参数（仓库地址及令牌）

注意：仓库地址不需要包含Gitee官网段

配置完成了，就可以开始即使图床了！

PicGo 上传图片并获取图片URL

通过图片URL您可以在您需要的任何地方加入并访问了！（注意：Gitee对访问文件大小有限制，小于1M）

Typora 使用 PicGo 图床

Typora 是一款轻便、简洁、免费的Markdown编辑器，支持即时渲染。

Typora 安装及设置

Typora官网下载，安装。

运行Typora，菜单【文件】》【偏好设置】进行图像的偏好设置

Typora 图床使用

在编辑区正文中插入图片，选择本地图片，Typora会自动上传至 Gitee 并转换成图片URL。

VSCode 使用 PicoGo

VSCode是微软开发的一款轻量、开源且功能强大的代码编辑器，支持跨平台。

VSCode 安装

VsCode官网下载，安装。

运行VSCode，并安装 PicGo 插件

VSCode 图床使用

默认 Ctrl+Alt+E 插入本地图片，插入后自动上传至 SM.MS(最大支持10M)，反应速度有点慢。
也可进入 VSCode 设置进行 PicGo 图床设置。

理正三维杆件分析进行框架结构计算

发表于 2022-06-09 更新于 2022-06-12

四层框架（梁柱结构）受风荷载作用的静力分析。

问题：

四层、长边二跨（2X6000mm）、宽一跨（1X5000mm）、梁柱均采用C40

一层，层高：4500mm，柱截面：600mmX600mm，梁截面：400mmX600mm

二~四层，层高：3000mm，柱截面：400mmX400mm，梁截面：200mmX600mm

计算长边风荷载作用下变形及内力。

分析软件：

采用理正CAD云或理正结构工具箱中的三维杆件进行结构分析。

计算分析步骤：

1. 建模

  • 绘制简图线

三维杆件支持软件内绘制，也可采用DXF文件导入，以DXF文件导入演示，先在AutoCAD等CAD软件中进行简图线绘图并另存为DXF文件（建议保存为R12版DXF文件），然后在三维杆件中进行导入DXF简图线。

  • 定义构件截面：点击截面定义，输入截面数量并进行截面定义（不考虑自重，混凝土容重输入0）

  • 布置构件：点击构件布置，选择截面，【确定】后在图上拾取简图线进行构件布置

  • 加载

点【荷载】，再点【荷载组合】进行荷载组合定义

点【布置荷载】，选择荷载类型及输入荷载值后在图上选择节点或构件进行荷载布置

风荷载演示中简化以节点集中荷载加在节点位置，也可以构件分布荷载加载在柱构件上。

2. 计算

点【计算】进行结构受力分析。

3. 结果查询

点【查询】，选择查询内容（位移、轴力、剪力、扭矩、弯矩、支座反力）及参数，点【重画】即可查看相应的分析结果。

左键双击右侧文本框中的内容，可进行左侧构件及节点等图上定位。

4. 计算书输出

点击【计算书】，再选择计算书操作菜单【生成计算书】，选择输出内容【确定】即可查看计算书。

柱下桩承台冲切的正确计算方法

发表于 2022-06-07 更新于 2022-06-12

以五桩承台算例分析阐明柱下桩承台冲切的正确计算方法。

问题及分析：

《地基规范》8.5.19及《桩基规范》5.9.7，冲切破坏锥体应自柱边可承台变阶外至相应桩顶边缘连线构成的锥体，锥体与承台底面夹角不小于45度。对某些情况下，满足上述条件的冲切锥可能存在多个，规范未明确说明需要计算哪个，所有很多工程师根据一般经验直接取最小破坏锥体来计算，但实际上最小破坏锥体可能不是最不利破坏锥体，应计算所有潜在破坏冲切锥，按最不利破坏冲切锥来进行设计。

柱对承台冲切，参见《桩基规范》（JGJ 94-2008）5.9.7：

设计实例

参见 刘金波 主编《建筑结构新规范系列培训读本 - 建筑桩基技术规范理解与应用》例9.3.2，五桩承台计算。

分析软件：

采用理正CAD云或理正结构工具箱中的独立桩承台。

计算分析步骤：

1. 按模型输入参数

由于只需要进行承台冲切计算，只需要输入以下红框内关键参数即可。

柱底荷载不括承台及覆土重，交互值按例9.3.2荷载值减承台及覆土重考虑。

钢筋合力点距离由于软件没有区分X、Y向配筋上下侧关系，按例题平均值（80+60）/2=70mm考虑。

2. 计算

点【计算】进行桩承台计算。

3. 结果查询及对比

点【结果查看】，选择【柱冲切】简图，点【详细】即可查看详细计算书结果

经比较柱冲切结果与例9.3.2冲切力是一致的，但抗冲切力不一致，分析主要是以下两个原因造成的：

1） 冲切锥取法不一样，例9.3.2中按最小冲切锥计算，程序按最不利冲切锥计算（主要原因）

2） 例9.3.2中X、Y向钢筋合力点至混凝土外边距离分别考虑

4. 手算分析对比

按下图三种潜在冲切破坏锥分别计算抗冲切力。

下表是三种潜在冲切破坏锥分别按钢筋合力点不同方式考虑的计算结果

从上表看按X、Y钢筋合力点分别考虑计算时，X向45度冲切时为最不利抗冲切力最小；X、Y向钢筋合力点按平均值考虑时X、Y向45度冲切结果与程序一致。

结论

通过以上分析柱下桩承台冲切应考虑满足规范冲切锥要求的潜在冲切锥并按最不利冲切锥考虑计算，在X、Y向钢筋合力点按相同值考虑时，理正独立桩承台计算结果是可信的。

理正结构工具箱载体桩多承台沉降计算使用

发表于 2022-06-06 更新于 2022-06-12

如下图，当需要计算CT-1~4在CT-1中点的沉降时，可以下列操作步骤进行：

  1. 设置载体桩CAD拾取配置及图纸准备

    • 如果需要拾取的图形是图块、自定义实体，则先在CAD中进行炸开操作

    • 查看CAD中需要拾取的承台及桩图形所在图层，并设置载体桩程序图层关键字及承台边界角点交互方式，上图中承台图层（1506）、桩图层（1507），设置如下：

  2. 在载体桩软件中点【拾取】进入CAD拾取承台（CT-1）及桩，并拾取沉降计算点，点【确认】，承台、桩及沉降计算点数据返回载体桩软件，再在载体桩软件中设置其它设计参数（如：荷载等），计算CT-1在沉降计算点沉降

  3. 重复第2步操作，分别拾取CT-2 ~ 4承台及桩，并拾取沉降计算点（与第2步为同一点）设置其它设计参数，计算CT-2 ~ 4在沉降计算点沉降

  4. 将CT-1 ~ 4在沉降计算点沉降求和，则可计算CT-1 ~ 4在沉降计算点的总沉降
